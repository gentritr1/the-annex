# Scene expansion plan

Checked against the current production surface on 2026-07-22.

## Direction

The Annex should expand as a sequence of physical civic places, not as a collection of decorative banners. Keep the current deterministic reducer and DOM controls. Scene media may interpret canonical state, but it never creates evidence, trust, access, alarms, outcomes, or run history.

The first shipped direction is dependency-free spatial staging:

- compact WebP environment plates;
- authored CSS perspective and a compensated four-plane diorama for district navigation;
- HTML controls and authored copy in normal document order;
- CSS/SVG state layers for selection, corroboration, alarm, and filing;
- the existing reduced-motion and high-contrast preferences;
- WebGL only when a mechanic genuinely needs spatial manipulation or occlusion.

## Acceptance gates for every scene

- Canonical game-state tests remain unchanged or expand only for authored cross-references.
- Essential evidence, cost, state, and instruction never exist only in artwork, color, audio, or motion.
- Keyboard, pointer, and touch reach the same existing action.
- Motion has a static reduced-motion composition; video does not mount or download in that mode.
- Large text and high contrast produce no horizontal overflow or clipped action.
- Generated art contains no readable accidental signage, unapproved likeness, logo, watermark, or franchise reference.
- `npm run lint`, `npm run test`, and `npm run build` pass.
